# AGENTS.md

## 프로젝트 개요

이 프로젝트는 Streamlit으로 만든 개인용 애니 시청 기록 앱이다. 사용자는 애니를 검색해서 목록에 추가하고, 시즌별/에피소드별로 어디까지 봤는지 체크한다. 모바일 WebView/APK와 Streamlit Cloud 배포 환경에서 쓰는 것을 전제로 한다.

처음에는 단순한 애니 체크 앱이었지만, 사용 중 불편했던 점을 계속 반영하면서 다음 방향으로 발전했다.

- 모바일에서 스크롤을 적게 하도록 화면을 compact하게 유지한다.
- 새로 나온 화가 있는 애니와 일반 목록을 분리한다.
- 하차 개념은 `보류`로 표현한다.
- 나중에 볼 애니는 `찜`에 따로 담는다.
- Streamlit Cloud 절전/리부트 후에도 데이터가 유지되도록 브라우저 localStorage와 백업 기능을 사용한다.
- 외부 API/TMDB/뉴스/RSS 데이터가 틀리거나 부족할 수 있으므로 더미값, 깨진 이미지, 관련 없는 뉴스는 최대한 숨긴다.

## 실행 방법

로컬에서 실행할 때:

```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py
```

이미 `.venv`가 있으면:

```powershell
.\.venv\Scripts\activate
streamlit run app.py
```

## 파일 분리 구조

기능 수정은 가능한 한 해당 모듈만 건드린다. 다른 기능이 갑자기 바뀌지 않게 공통 파일 수정은 최소화한다.

- `app.py`: Streamlit 화면 흐름, 세션 상태 연결, 화면 렌더링 진입점
- `anime_checker/ui_assets.py`: 전체 CSS, 모바일 레이아웃, 상단 이동 링크
- `anime_checker/menu_library.py`: `새 화`, `목록`, `보류`, 편성표 카드와 상세 진입 버튼
- `anime_checker/menu_wish.py`: `찜` 카드, 목록 추가, 찜 삭제 버튼
- `anime_checker/menu_new_anime.py`: `신작 애니` 카드, 찜/추가 버튼
- `anime_checker/menu_news.py`: `애니 소식` 목록 카드와 상세 진입 버튼
- `anime_checker/back_handler.py`: 폰 자체 뒤로가기 처리 JS
- `anime_checker/news.py`: 애니 소식 RSS 수집, 뉴스 필터링, 이미지 추출
- `anime_checker/storage.py`: 백업 JSON 구조, 파일 저장/불러오기

수정 요청을 받으면 관련 모듈부터 고치고, 수정 후에는 최소 `python -m py_compile app.py anime_checker/*.py` 검증을 실행한다. 앱 동작에 영향이 있는 변경이면 로컬 Streamlit 실행까지 확인한다.

## API 키

TMDB API 키는 Streamlit secrets로 넣는다.

로컬에서는 `.streamlit/secrets.toml`에 아래처럼 둔다.

```toml
TMDB_API_KEY = "your_tmdb_api_key"
```

주의:

- `.streamlit/secrets.toml`은 GitHub에 올리지 않는다.
- TMDB API 키가 없으면 검색/애니 추가/시즌 정보/포스터/극장판 정보/신작 애니 목록이 제대로 동작하지 않을 수 있다.
- `애니 소식`은 Google News RSS 기반이라 TMDB API 키 없이도 동작해야 한다.

## 데이터 저장 방식

현재 앱의 핵심 데이터는 다음 두 곳에 저장된다.

- 브라우저 localStorage: 실제 사용자 데이터 유지용
- `anime_check_data.json`: 로컬/서버 보조 저장용

중요한 의도:

- Streamlit Cloud는 절전/리부트가 발생할 수 있으므로 서버 파일 저장만 믿으면 안 된다.
- 같은 폰/같은 브라우저/같은 앱 주소라면 localStorage에서 목록과 시청 기록이 복구되어야 한다.
- 앱 데이터/사이트 데이터 삭제, 앱 삭제 후 재설치, 다른 브라우저 사용, 주소 변경 시 localStorage 데이터가 사라질 수 있다.

백업 기능:

- `목록` 탭에 `백업 / 불러오기` 영역이 있다.
- 파일 다운로드/업로드 방식이 있다.
- 모바일 WebView에서 파일 업로드/다운로드가 안 먹을 수 있어서, 백업 JSON 내용을 복사하고 다시 붙여넣어 복원하는 텍스트 방식도 제공한다.
- 백업/불러오기는 `새 화`, `목록`, `보류`, `찜`, 시청 기록을 함께 저장하고 복원해야 한다.
- 백업 적용 전에는 전체 개수, 보류 개수, 찜 개수, 시청 기록 개수를 사용자에게 보여준다.

## 주요 탭 구조

메인 화면 탭은 다음 순서를 유지한다.

- `새 화`
- `목록`
- `보류`
- `찜`
- `신작 애니`
- `애니 소식`

### 새 화

- 새로 방영된 화가 있고 아직 시청 완료하지 않은 애니만 표시한다.
- 새 화가 없는 애니는 `목록` 탭으로 빠진다.
- `보류`된 애니는 새 화 표시에서 제외한다.
- 편성표도 `새 화` 탭에 같이 보여준다.

### 목록

- 일반 시청 목록이다.
- 새 화가 없는 애니, 시청 완료된 애니 등이 표시된다.
- 검색 버튼은 `목록` 제목 오른쪽에 작게 둔다.
- 총 개수는 목록 영역 오른쪽 아래 느낌으로 작게 표시한다.
- `백업 / 불러오기` 영역이 있다.

### 보류

- 예전의 `하차` 탭을 `보류`로 이름만 바꾼 것이다.
- 내부 데이터 키는 여전히 `dropped`를 사용할 수 있다.
- 보류된 애니는 새 화 알림/편성표에서 제외한다.
- 상세 화면의 버튼은 `보류` / `복귀`로 표시한다.

### 찜

- 검색 결과나 신작 애니에서 `찜` 버튼으로 담은 애니를 보여준다.
- 찜 목록에서는 `목록 추가`와 `삭제`를 할 수 있다.
- 목록에 추가하면 찜 목록에서는 제거된다.

### 신작 애니

- TMDB 데이터를 가져오되 한국 OTT에서 서비스 중이거나 서비스 예정인 애니 중심으로 보여준다.
- 일본어 제목만 있는 항목은 최대한 숨기고 한국어 제목이 있는 항목 중심으로 보여준다.
- `추가` 버튼과 `찜` 버튼은 같은 줄에 작게 배치한다.
- 최근 방영작뿐 아니라 가까운 방영 예정작도 같이 확인할 수 있게 한다.
- `신작 애니` 제목 옆에는 `YYYY.MM.DD. HH:MM 기준` 형식으로 데이터를 가져온 기준 시각을 표시한다.
- 오늘 기준 미래 방영일이면 `방영일: YYYY.MM.DD. 방영예정`으로 표시한다.
- 포스터가 없으면 더미 이미지를 넣지 말고 이미지 영역을 숨긴다.
- 카드가 너무 커지지 않게 유지한다.

### 애니 소식

- Google News RSS를 이용해 애니 방영 예정, 신작 공개일, 시즌 발표, 극장판 개봉 중심의 소식을 가져온다.
- 사용자가 원하는 애니 소식의 핵심 주제는 다음이다.
- 신작 애니 방영/공개/공개일/방영일
- 새로운 깃수, 새 시즌, 2기/3기/후속작/속편 제작 발표
- 극장판 개봉, 재개봉, 특별상영
- 극장판/애니메이션의 인기, 흥행, 박스오피스, 관객 수, 순위
- `최신 애니 소식` 제목 옆에 `YYYY.MM.DD HH:MM 기준` 형식으로 앱이 뉴스를 가져온 기준 시각을 표시한다.
- 뉴스는 앱을 켰을 때 한 번 가져오는 느낌으로 유지한다. 너무 자주 갱신하지 않는다.
- 관련 없는 뉴스, 해외 소스, 웹툰/게임/굿즈/리뷰성 글은 최대한 제외한다.
- 단, 인기/흥행/박스오피스/순위/랭킹은 사용자가 보고 싶어 하는 주제이므로 무조건 제외하지 않는다.
- 이미지가 실제로 로딩 가능한 기사 이미지일 때만 보여준다.
- 기사 이미지는 가능하면 앱 서버가 먼저 bytes로 받아서 보여주며, 브라우저 hotlink 때문에 생기는 액박을 줄인다.
- 더미 이미지나 깨진 이미지가 나오면 이미지 영역 자체를 숨기는 방향이 좋다.
- 뉴스는 관련도 점수를 매겨 신작/새 시즌/극장판/재개봉/흥행/관객 수/순위 기사부터 남긴다.
- 모바일에서 뉴스 카드가 찌그러지지 않도록 한 줄에 한 기사 카드로 보여주는 방향을 유지한다.

## 애니 추가/검색

검색 결과 카드에는 다음 버튼이 있다.

- `목록 추가`
- `찜`

요구사항:

- 버튼은 너무 크게 한 줄을 다 먹지 않게 한다.
- 가능하면 같은 줄에 작게 정렬한다.
- 이미 목록에 있으면 `목록에 있음` 또는 `완료/추가됨` 상태로 표시한다.
- 찜 상태이면 `찜해제`로 바뀐다.

## 상세 화면

상세 화면에는 작품 상태와 최근 시청 기록이 compact하게 표시된다.

표시 예:

```text
매주 수요일
최근 2기 8화
```

현재 버튼:

- `보류` / `복귀`
- `삭제`

예전에 있던 `정보` 버튼은 제거했다. 나무위키/외부 링크가 모바일 WebView에서 자주 깨지고, UX가 좋지 않았기 때문이다.

## 시즌/깃수 처리

요구사항:

- TMDB에서 시즌이 제대로 나뉘지 않는 작품도 있다.
- 주술회전처럼 전체 화수가 1화부터 쭉 이어지는 작품은 날짜 간격/수동 규칙으로 시즌을 분리한다.
- 지옥락처럼 시즌이 나뉘지 않는 사례도 자동 분리 대상으로 본다.
- 0부작, `9999.12.31` 같은 placeholder 시즌은 제거한다.
- 아직 안 나온 시즌이 0부작으로 나오면 보여주지 않는다.
- 시즌이 바뀌면 포스터도 가능하면 시즌 포스터로 바뀌어야 한다.
- 시즌 이미지가 더미/Unsplash이면 작품 대표 포스터로 대체한다.

주의:

- 수동 규칙을 작품별로 계속 늘리는 것보다, 방영일 간격 기반 자동 분리를 우선한다.
- 그래도 특정 작품은 예외 규칙이 필요할 수 있다.

## 에피소드 체크 화면

요구사항:

- `시청` 버튼을 누르면 `완료`로 바뀐다.
- `완료`를 누르면 다시 `시청`으로 바뀐다.
- 시청/완료 버튼 색상은 다르게 표시한다.
- 체크박스 방식은 사용하지 않는다.
- 버튼은 오른쪽 끝에 작게 둔다.
- `1화 : 제목`과 `방영일` 두 줄의 가운데 오른쪽에 버튼이 정렬되게 한다.
- 에피소드 사이에는 살짝 공백 또는 희미한 구분선을 둔다.
- `에피소드 리스트` 같은 불필요한 문구는 제거한다.
- 시즌 안에서 화수 체크하는 화면 상단에는 `뒤로가기`와 `메인화면` 버튼이 있다.

시청 로직:

- 어떤 화를 `시청`으로 바꾸면 그 이전 화들도 함께 시청 처리한다.
- 어떤 화를 `완료`에서 다시 해제하면 그 화 이후를 함께 미시청 처리한다.

## 극장판/영화

요구사항:

- 일반 애니 시즌 목록과 섞지 않고, 시즌 목록 아래에 `극장판/영화` 섹션으로 따로 보여준다.
- 내용 설명은 길게 보여주지 않는다.
- 제목, 개봉일, 러닝타임을 표시한다.
- `시청` / `완료` 버튼을 일반 에피소드와 비슷하게 작게 표시한다.
- `정보` 버튼은 제거했다.
- 극장판 시청 버튼을 눌렀을 때 앱이 새로 열리는 문제가 있었으므로 일반 버튼 토글 방식만 사용한다.

## 모바일/뒤로가기 UX

앱은 폰에서 쓰는 것을 중요하게 본다.

요구사항:

- 메인 화면에서만 뒤로가기 두 번으로 종료한다.
- 다른 화면에서 폰 자체 뒤로가기를 누르면 상위 화면으로 이동해야 한다.
- 시즌 선택 화면에서 뒤로가기 시 앱 종료 안내가 뜨지 않고 메인 화면으로 돌아가야 한다.
- 에피소드 체크 화면에서 폰 뒤로가기를 누르면 시즌 선택으로 돌아간다.
- `이전 화면으로 돌아왔습니다` 같은 안내 문구는 표시하지 않는다.
- Streamlit 오른쪽 아래 배지/왕관이 버튼을 가리지 않도록 아래 여백을 둔다.

## UI/레이아웃 원칙

전체적으로 모바일에서 스크롤이 길어지지 않게 한다.

- 공백은 작게 유지한다.
- 제목 글자 크기는 과하지 않게 한다.
- 카드 안 버튼이 한 줄 전체를 먹지 않게 한다.
- 가능한 버튼은 같은 줄 오른쪽에 작게 배치한다.
- 애니 제목 글자 크기는 너무 크지 않게 한다.
- 장르/방영일 줄간격은 너무 붙지 않게 한다.
- 긴 제목이 모바일에서 옆으로 넘치지 않게 `word-break`, `overflow-wrap` 등을 사용한다.
- 카드 안에 또 카드처럼 보이는 불필요한 네모 박스를 만들지 않는다.

## 뉴스 이미지/기사 필터 주의사항

애니 소식에서 자주 문제가 있었다.

문제:

- 더미 이미지가 들어옴
- 깨진 이미지/액박이 보임
- 기사 원문에는 이미지가 있는데 앱에서 못 가져옴
- 해외 기사나 번역된 외국어 기사, 애니와 관련 없는 글이 들어옴
- 원문 기사 보기 버튼이 구글 아이콘 같은 이상한 페이지로 감

현재 방향:

- Google News 링크를 원문 기사 링크로 해석한다.
- 한국 도메인 또는 한국어 출처 중심으로 필터링한다.
- 기사 HTML에서 `og:image`, `twitter:image`, JSON-LD, `data-src`, `srcset` 등을 찾아 이미지를 추출한다.
- 이미지 URL이 실제 `image/*` 응답일 때만 보여준다.
- 원문 이미지 URL을 브라우저에 그대로 맡기지 않고 bytes로 받아서 표시하면 hotlink 액박을 줄일 수 있다.
- 뉴스는 단순 키워드 포함 여부만 보지 말고 관련도 점수로 정렬한다.
- 실패하면 더미 이미지를 보여주지 말고 이미지 영역을 숨긴다.

## Streamlit Cloud 절전 관련

Streamlit Cloud 앱은 일정 시간 접속이 없으면 절전 상태가 될 수 있다.

요구사항:

- 절전 후 앱을 깨워도 데이터가 사라지면 안 된다.
- 서버 파일만 믿지 말고 localStorage에서 복구해야 한다.
- 리부트 후에도 같은 폰/같은 브라우저라면 데이터가 복구되어야 한다.

주의:

- localStorage는 같은 브라우저/같은 앱 주소 기준이다.
- 사용자가 앱 데이터/사이트 데이터/브라우저 데이터를 지우면 사라질 수 있다.
- 그래서 백업 저장/붙여넣기 복원 기능을 유지한다.

## 2026-05-31 안정화 메모

이번 수정에서 메인 탭 전환은 HTML 링크가 아니라 Streamlit 버튼 기반 메뉴로 바꿨다. `새 화`, `목록`, `보류`, `찜`, `신작 애니`, `애니 소식`은 3개씩 2줄로 보이고, 버튼을 누르면 `st.session_state.main_section`이 직접 바뀐다. 앞으로 메인 메뉴를 고칠 때는 단순 `<a href="?main_nav=...">` 방식으로 되돌리지 않는다. Streamlit 상태와 URL query가 어긋나 탭이 눌리지 않는 문제가 생긴다.

기기 자체 뒤로가기는 JS handler version 17 기준이다. handler는 `components.html` iframe에서 `parent`, `top`, 현재 `window`에 순서대로 `eval` 주입한다. 현재 DOM의 `anime-current-view` 상태를 우선하되, 폰 WebView에서 DOM 상태를 놓치는 경우를 줄이기 위해 `popstate` event state의 하위 화면 상태도 fallback으로 사용한다. WebView가 `history.pushState` 항목을 뒤로가기 대상으로 보지 못하는 경우가 있어 `#anime-checker-guard-v17` hash guard와 `hashchange` handler도 함께 사용한다. 폰 자체 뒤로가기에서는 Streamlit 버튼 자동 클릭보다 `app_back` query fallback을 우선해 상위 화면을 복구한다. 화면에 보이는 `뒤로가기` 버튼은 사용자가 직접 누르는 버튼으로 유지한다. 매 렌더링마다 history guard를 다시 보강해서 메인 탭에서 뒤로가기 시 이전 상세 화면 history로 튀지 않게 한다. 에피소드 화면에서 뒤로가기 시 시즌 선택으로, 시즌 선택에서 뒤로가기 시 메인으로 돌아와야 한다. 메인 화면에서만 두 번 뒤로가기 종료 안내가 떠야 한다.

신작 애니는 TMDB API 키가 없을 때 데모 데이터가 중복 key를 만들지 않도록 page 1에서만 demo item을 반환한다. 실제 신작 목록도 id 기준으로 중복 제거 후 렌더링한다. `찜`/`추가` 버튼 key에는 index를 함께 넣어 중복 key를 피한다.

애니 소식은 여러 뉴스 피드와 기사 이미지 추출 때문에 오래 멈출 수 있어 시간 예산을 둔다. 뉴스 수집이 느리면 앱이 계속 스피너에 묶이지 않도록 빠르게 간단 목록 또는 안내 카드로 빠져야 한다. 뉴스 카드는 모바일에서 찌그러지지 않게 한 줄에 한 카드로 유지한다.

로컬 검증 결과:

- `python -m py_compile app.py` 통과
- 메인 메뉴 `새 화`, `목록`, `보류`, `찜`, `신작 애니`, `애니 소식` 실제 클릭 전환 확인
- 검색 결과 표시와 목록 추가 확인
- 상세 화면 진입 확인
- 에피소드 `시청` / `완료` 토글 확인
- 극장판/영화 `시청` / `완료` 토글 확인
- 에피소드 화면에서 기기 뒤로가기 흐름: 에피소드 -> 시즌 선택 확인
- 시즌 선택 화면에서 기기 뒤로가기 흐름: 시즌 선택 -> 메인 확인

추가 메모: 현재 뒤로가기 JS는 실제 실행이 필요하므로 `streamlit.components.v1.html(..., height=1)`을 사용한다. scroll JS처럼 1회성인 경우만 `height=0`을 쓸 수 있다. `st.html(..., unsafe_allow_javascript=True)`는 `<script>`가 DOM에 남아도 실행되지 않는 사례가 있어 뒤로가기 handler에는 쓰지 않는다.

## 2026-05-31 재수정 메모

- 메인 메뉴는 `st.pills`를 사용한다. `st.columns` 버튼 메뉴는 모바일에서 버튼 하나가 한 줄씩 먹을 수 있으므로 다시 쓰지 않는다.
- 폰 자체 뒤로가기는 메인 화면이 아닌 하위 화면에서 `app_back` query fallback을 먼저 사용한다. JS handler version은 17이다.
- 메인 화면에서는 `새 화`가 아닌 메인 탭에서 뒤로가기 시 `새 화`로 돌아가고, `새 화`에서만 뒤로가기 두 번으로 종료 흐름을 허용한다.
- 애니 소식은 enrich 기사만 반환하지 말고 RSS fallback 기사로 `max_items`까지 채운다. 한두 개만 보이는 상태로 만들지 않는다.
- 신작 애니는 `popularity.desc`가 아니라 `first_air_date.desc` 기준이다. 한국 지역 OTT 결과는 1~8페이지까지 가져오고, provider 데이터가 늦게 갱신되는 문제를 줄이기 위해 한국어 제목이 있는 최신 일본 애니 후보도 1~4페이지까지 합쳐 중복 제거한다.
